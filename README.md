# CodeAlpha_Secure_Coding_Review

## 🛡️ Overview
This repository contains my submission for the **Secure Coding Review** task as part of the Cyber Security Internship at **CodeAlpha**. The objective of this project is to perform a static code analysis on a vulnerable application, identify critical security flaws, and provide actionable remediation steps.

## 🎯 Target Application
**PyGoat**: An intentionally vulnerable web application (Python/Django) used to demonstrate common security flaws, often aligning with the OWASP Top 10.

## 🛠️ Tools Used
* **Bandit**: A Python-based Static Application Security Testing (SAST) tool used to automatically find common security issues in Python code.
* **Manual Inspection**: To verify automated findings and document exact remediation steps.

## 🚨 Key Vulnerabilities Identified
During the code review, multiple vulnerabilities were discovered. The detailed report focuses on the following critical and medium-risk findings:
1. **Weak Password Hashing:** Use of the outdated MD5 algorithm (CWE-327).
2. **OS Command Injection:** Unsafe use of `subprocess.Popen` with `shell=True` (CWE-78).
3. **Debug Mode Enabled:** Application configured to run in debug mode, exposing sensitive data (CWE-94).
4. **SQL Injection:** Unsafe string concatenation in database queries (CWE-89).

## 📁 Repository Contents
* `bandit_report.html`: The raw automated scan report generated by Bandit.
* `Secure_Coding_Review_Report.pdf`: The final, detailed technical report documenting the findings, their severity, and recommended fixes.
* `pygoat/` (or zipped source): The source code of the analyzed application.
